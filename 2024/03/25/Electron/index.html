安装electron

-D 是 pnpm add 命令中的一个选项，它表示你希望将 electron 包作为开发依赖安装。

意思是electron包将会被添加到devDependencies中，而不是dependencies中。

开发依赖通常是指在开发过程中需要用到的工具、库或框架，但在生产环境中并不需要。

修改package.json文件中的内容，如下：

  • name 作为app.name的默认值
  • productName 字段会做为app.name的默认值，优先级高于name
  • main 主进程的入口文件
  • dev 作为项目的启动命令

多进程模型

electron将使用两种类型的进程：主进程 和 渲染进程。

主进程

  • 调用原生Node.js的API，来实现系统级别的操作（文件的操作等）。
  • 通过原生菜单改变渲染进程页面内容。
  • IPC通信使用 ipcMain 和 ipcRenderer两个模块传递消息。

渲染进程

通过BrowserWindow打开的窗口生成一个单独的渲染器进程。

预加载脚本

主进程和渲染进程之间的通信，是通过preload.js预加载来实现的，简单来说，如果想要用网页直接操作系统文件，网页直接调用node模块，就要使用到这个预加载脚本。
