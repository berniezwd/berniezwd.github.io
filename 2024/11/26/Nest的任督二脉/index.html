基础概念扫盲

进门前先了解下基本概念

概念 描述 作用
controller 控制器 处理路由、解析请求参数
handler - 控制器内处理路由的方式
service 服务层 实现业务逻辑（操作数据库）
dto data transfer object 数据传输对象 控制请求体中的数据对象
module 模块（用户模块、支付模块） 项目模块化
entity 实体 映射数据库中表的字段
ioc inverse of controller 反转控制（依赖注入） 声明依赖后，nest会自动注入依赖的实例
aop aspect oriented programming 面向切面编程 在多个请求响应中插入可复用的逻辑，比如日志记录、请求拦截、响应拦截等，包含：middleware、interceotor、guard、exception、filter、pipe
nest cli 脚手架 快速创建项目、模块、controller、service等

命令 描述 结果
–flat 生成对应的目录 src/xxx/xxx.controller.ts

打包

命令选项：

–webpack ：使用webpack方式进行打包成单模块，可以显著提升加载性能。

–tsc：这是默认的打包方式，这里的打包实际是编译ts，并不会进行代码的打包。

命令选项：

–watch：改动文件后自动重新build，省去重启服务的操作。

–debug：启动调试模式。

–exec：指定runtime，默认是用node运行。

所有的脚手架命令选项都可以在nest-cli.json中进行配置

我们分析下provider注入的过程；

  1. 首先在service中，使用@Injectable装饰器对AppService进行修饰；

  3. 在app的controller层中，声明这个provider后，就可以使用这个AppService实例了；

全局模块

模块导出provider（即使用@Injectable装饰的类），另外一个模块需要在模块中imports，才可以使用别的模块的provider。

  1. A模块将AService导出（导出的是Aservice）；

  2. B模块将AModule导入（导入的是整个A模块）；

如果被导出的模块，在很多模块中都被使用到，需要一个个模块中imports，是一件很繁琐的事情！

所以可以将公共的模块，使用@Global装饰器设置成全局模块。（全局模块缺点：不好追溯provider，降低代码可维护性）

AOP

Aspect Oriented Programming 面向切面编程；

前端发起请求到后端接收：经过controller（参数接收）、service（逻辑处理）、repository（数据库访问）；

AOP的作用就是在上面的链路中，透明的加上一些任务，例如：日志记录、权限控制、异常处理等；

这样处理既可以保持业务逻辑的纯粹性，逻辑又可以复用，还可以动态的增删；

上面的示例展示了怎么样动态注册一个模块，其实上面示例中register方法名可以是自定义的，但是Nest约定了三个方法名：

  1. register：用一次模块传一次配置。
