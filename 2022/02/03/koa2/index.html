命令 解释

介绍

学习Koa需要了解到洋葱模型:

Express和Koa作为同样作用的中间件，两者的执行顺序不太一样；

在express框架中，中间件是顺序执行，通过next()逐级匹配，然后响应结果

Koa2中，从第一个中间件开始执行，遇到next()进入下一个中间件，一直执行到最后一个中间件，再逆着顺序继续执行上一个中间next()方法后面的代码,一直到第一个中间件执行结束才完成响应;

什么是中间件？

中间件是用于匹配路由之前或者匹配路由之后所完成的一些列操作;

在express中间件是一个函数，他可以访问请求对象request或者响应对象response，和 web 应用中处理请求-响应循环流程中的中间件，一般被命名为 next 的变量

中间件功能：

  • 执行任何代码
  • 修改请求和响应对象
  • 终结请求-响应循环
  • 调用堆栈中的下一个中间件

如果我的 get 、post回调函数中，没有 next 参数，只会匹配上第一个路由，就不会往下匹配了。如果想往下匹配，就需要 next（）。

中间件优先级：

中间件 解释
router.get() 路由中间件
app.use() 应用级中间件(优先级更高)

当Koa2中有应用级中间件和路由级中间件同时存在的时候，会先执行应用级中间件;

中间件分类：

  • 应用级中间件

有next()的情况下，就会继续向下匹配路由，

没有next()的情况，就不会向下继续匹配了。

  • 错误处理中间件

此处的执行流程：

  1. 先全局匹配路由，打印 这是中间件01;
  2. 然后遇到next()执行下一个中间件，找到news路由 打印这是新闻02;
  3. 匹配不到其他路由后，再返回来执行app.use里面 next 之后的代码，最终响应app.use的执行结果;
  • 路由级中间件

匹配两次news路由，如果第一次匹配的news没有next，就不会继续执行了

Koa路由

koa中的路由和Express有所不同，在Express中直接引入Express就可以配置路由，但是在Koa中我们需要安装对应的koa-router路由模块来实现。

路由基本使用

  1. 引入koa模块和koa-router模块
  2. 实例化koa和router
  3. 配置路由
    1. ctx包含了request、response等信息
    2. ctx.body是渲染页面数据，相当于原生里的response.end()
  4. 启动路由，让路由生效
  5. router.allowedMethods() 官方建议配置，不配置也不会报错

get传值

在koa2中get传值通过request接收，方法有两种：query 和 querystring

  1. query ：返回参数对象（也包含了域名信息、当前url信息等）。
  2. querystring：返回的是请求字符串。
