常见的全局对象

常见的全局对象官网参考

常见的特殊全局对象，不能在命令行交互中使用，包括：__dirname、__filename、exports、module、require等。

  1. __dirname：当前目录的路径
  2. __filename：当前文件的路径

process提供了进程相关的信息

比如Node运行环境、参数信息、传递给node的参数等。

比如：在运行某个脚本index.js 中，终端运行命令node index.js zwd nickname=bernie

  1. console.log() node输出信息

  2. console.clear()清除打印台输出的信息

  3. console.trace()打印函数调用栈（看函数在哪些地方被调用）

  4. console.table(object)以table形式打印对象，左边是键名，右边是键值

setTimeout：延迟器

setInterval：定时器

setImmediate：立即执行

CommonJs模块化

Node是CommonJs在服务器一个具有代表性的实现；

在Node中每一个js文件都是一个单独的模块；

CommonJs规范的核心变量：exports、module.exports、require

exports和module.exports可以负责对模块中的内容进行导出；

require函数可以导入其他模块(自定义模块、系统模块、第三方库模块)中的内容；

模块化

exports是一个对象; 本质是对象的引用赋值；

exports.name = name 等价于 exports对象中的name属性的值是name值

或者使用解构的方式导入：

补充知识：（对象的赋值，实际是内存地址的赋值）

对象的赋值，内存中的表现为：

  • 在堆内存中分配空间，用于保存{ name:'bernie', age: 18 }这个对象的键值对，这个空间对应有一个内存地址，假设为0x100，obj实际保存的是对象的内存地址const obj = 0x100
  • 而以上的赋值操作，实际上是对obj内存地址的引用info = 0x100

exports的导出能力，实际上是通过module.exports来实现的；

exports == module.exports

  • 以上会在堆空间重新分配一个新空间，会有一个新的内存地址;
  • 在修改引用类型的值 ，并不会影响到导出源的对象的值；

exports和module.exports的区别？

exports导出的对象，无论在哪一个地方修改了值，都会互相影响；

module.exports导出的对象，无论在哪里修改值，都不会互相影响；

require函数，可以引入一个文件（模块）中的导出对象；

require的查找规则：

  1. 优先查找核心模块：fs、path等核心模块，一旦查找到，将会停止往后查找；
  2. require(X)：./ 、 ../ 、/（根目录）；
  3. 如果有后缀名，按照后缀名的格式查找
  4. 如果没有后缀名，会按照以下顺序查找：
    1. 直接查找文件X
    2. 查找X.js文件
    3. 查找X.json文件
    4. 查找X.node文件
  5. 如果没有对应的文件，将X作为一个目录

使用默认导出关键字是：export default；

默认一个模块只能导出一个，export default只能出现一次；

但是使用require()是可以执行的：（因为require本质是一个函数，if(flag)运行中，相当于是在执行调用require函数）

最好的推荐办法

方法一：（执行函数）

有时候真的需要根据条件或者动态导入模块，可以使用require()

方法二：（import函数，本质是被webpack调用）

webpack在遇到import()函数的时候，它会将它打包成好几个js文件，当页面有使用到该js文件的时候，才加载对应的模块；

import函数是异步加载，并且有返回值，返回Promise；

模块的加载过程

  • 模块在被第一次引入时，js代码会被运行一次。

  • 模块中的代码执行是同步的

  • 模块多次被引入，会缓存，最终只会加载运行一次。

events模块

events的延伸就是vue中的事件巴士，通过events模块，监听某个事件，发射某个事件。

在events模块中导入一个类，通过实例化这个类，可以得到on、off、emit等方法。

Package.json

必填项目：项目名称name、项目版本version

入口文件”main”

对于前端项目来说没什么意义，所有入口的文件都由webpack处理，无需这里的入口文件。

真正使用的场景，是将此项目作为工具包发布出去之后，别人想引用你这个包，那入口文件就是这里声明的位置，例如axios是这么声明入口位置的：

“script”

配置脚本名称功能。

在内置运行命令中，有几个关键字的命令run是可以省略的：start、stop、test、restart

本来是需要执行npm run start ，可以直接省略为 npm start

“dependencies”

指定无论开发环境还是生产环境都需要依赖的包

例如npm install axios

“devDependencies”

一些包在生产环境是不需要的，比如webpack、babel

通过命令--save-dev来讲依赖包安装在开发环境

例如npm isntall axios --sava-dev、npm install axios -D

“version”

版本号 版本说明
2.0.3 遵循（X，Y，Z）原则，X主版本号、Y次版本号、Z修订号
^x.y.z 表示x是保持不变的，y和z永远安装最新版本
~x.y.z 表示x和y保持不变，z永远安装最新的版本
