vscode 编译.ts 文件

  1. 在项目中执行tsc --init命令，生成tsconfig.json的配置文件。
  2. 在tsconfig.json文件中，outDir是指编译出来的 js 文件位置。
  3. vscode 菜单 => 终端 => 运行任务 => Typescript => tsc:监视-tsconfig.json。
  4. 每次修改 ts 文件，都会将 ts 文件自动编译为 js 文件到指定目录下。

TypeScript 数据类型

基础类型

基础类型包含有：boolean、string、number、null、undefined、symbol(Es6)、BigInt(Es10)

void 空类型

void 可以定义undefined和null类型。

在 TypeScript 中，可以用void表示没有任何返回值的函数（不包含返回 undefined 和 null）。

值得注意：

在非严格模式下，void 才可以定义 null 类型。

undefined、null是所有类型的子类型，在非严格模式下，是可以用其他类型进行分配的。

any 类型

any 是所有类型的顶级类型，不会进行类型检查。

声明一个变量，不指定类型的情况下，默认是any类型。

unkonwn 类型

unknown也被视为所有类型的顶级类型，和 any 一样可以被其他任意类型分配。

unknown可以赋值的对象只能是unknown和any

unknown类型不能赋值给其他类型。

小技巧：

当你要使用 any 的时候，不妨试试使用 unknown 类型。

nerver 类型

never类型代表那些永不存在的值的类型。

这也说明：never 类型的变量只能被 never 类型的变量所赋值。

interface 接口

定义对象类型要使用interface关键字，这类对象类型称为接口。

接口是一种约束，让对象的结构满足接口的格式。

  1. 重名的接口是可以进行合并的

  2. 接口可以被继承

函数重载

Java中的方法重载，指两个或两个以上同名函数，但他们的参数不一样，这时会出现函数重载的情况。

TypeScript中的重载，通过为一个函数提供多个函数类型定义，来实现多种功能的目的。

在ES5中，同名的函数或者变量，后者会覆盖前者

TypeScript 类

类的定义

构造函数constructor会在实例化类的时候触发。

通过static声明的方法，可以在实例创建前调用。

需注意：静态方法中无法直接调用类的属性，若要使用类的属性，需要将属性设置为静态属性，通过static定义

ES5 定义类是试用函数原型链的方式

类的继承

类的继承，通过关键字extend实现，父类被称为基类或者超类

类的继承包含两个关键字 constructor和super

  • constructor：构造函数会在实例化的时候触发
  • super：初始化父类的构造函数(调用父类的构造函数)
  • 若父类和子类有同名的方法，会就近原则，执行子类里的方法

类的属性修饰符

类的三种修饰符：public 、protected、private

public公有：类内部、子类、类外部都可以访问这个属性。

protected保护类型：类内部和子类可以访问，在外部不可以访问。

private私有：类内部可以访问，子类和外部都不可以访问。

类的多态

多态属于继承

父类定义一个方法不去实现，让继承它的子类去实现，每一个子类有不同的表现

父类中有eat方法，但是它自身不用，让Dog Cat两个子类重写 eat 方法，达到两者使用效果不一样，这就是多态

抽象类

抽象类，是作为父类的角色，不能直接被实例化。

通过关键字abstract来定义抽象类。

抽象类中的方法不能够调用实现，只能在派生类(子类)中实现。

泛型就是解决 类 接口 方法的复用性、以及对不特定数据类型的支持

TypeScript 命名空间

代码量大的情况下，避免变量命名冲突，可以将相似功能的函数、类、接口等放置到命名空间内

TypeScript 的命名空间可以将代码包裹起来，只对外暴露需要在外部访问的对象，命名空间内的对象通过

命名空间和模块的区别：

命名空间：内部模块，主要用于组织代码，避免命名冲突。

模块：ts 的外部模块的简称，侧重代码的复用，一个模块里可能有多个命名空间

TypeScript 装饰器

装饰器：是一种特殊类型的声明，它能够被附加到类声明，方法，属性或参数上，可以修改类的行为。

通俗讲装饰器就是一个方法，可以注入到类，方法，属性参数上来扩展、属性、方法、参数的功能。

常见的装饰器有：类装饰器、属性装饰器、方法装饰器、参数装饰器

装饰器的写法：普通装饰器（无法传参）、装饰器工厂（可传参）

若要启用实验性的装饰器特性，你必须在命令行或tsconfig.json里启用experimentalDecorators编译器选项：

不传参的时候，装饰器参数就是类

类装饰器（传参）

传参的时候，装饰器函数的参数就是传入的参数

类装饰器不仅可以拓展方法和属性，也可以替换HttpClint类原来有的构造函数：

类装饰器表达式会在运行时当做函数被调用，类的构造函数作为唯一的参数。

如果类装饰器返回一个值，他会使用提供的构造函数来替换类的声明。

需要注意：通过这种方式重载构造函数的时候，其他成员也要一起被重载，好比getData方法也需要在装饰器内重载。

属性装饰器

属性装饰器表达式会在运行时当作函数被调用，传入以下两个参数：

  1. 对于静态成员来说是类的构造函数，对于实例成员是类的原型对象
  2. 属性的名字

通过属性装饰器来修改类里面的属性值（给某个类中的属性设置默认值）

方法装饰器

方法装饰器和属性装饰性有点像，可以通过方法装饰器来扩展类的属性，以及修改类里的方法

方法装饰器传入 3 个参数：

  1. 对于静态成员来说是类的构造函数，对于实例成员来说是类的原型对象
  2. 当前方法的名称
  3. 成员的属性描述符(获取方法需要用.value)

tsconfig.json

compilerOptions 支持很多选项，常见的有 baseUrl、 target、baseUrl、 moduleResolution 和 lib 等。

compilerOptions 每个选项的详细说明如下：

  • 用于标识 TypeScript 项目的根路径；
  • 用于配置 TypeScript 编译器；
  • 用于指定编译的文件。
  • files - 设置要编译的文件的名称；
  • include - 设置需要进行编译的文件，支持路径模式匹配；
  • exclude - 设置无需进行编译的文件，支持路径模式匹配；
  • compilerOptions - 设置与编译流程相关的选项。
