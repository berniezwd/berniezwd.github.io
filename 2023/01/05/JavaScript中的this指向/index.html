JavaScript中的this指向

发表于2023-01-05|更新于2023-01-05|JavaScript

理解this

this是在函数调用时被绑定的一个对象

为什么使用this

下面通过对象字面量的形式创建出来一个对象，当我们调用对象内预置的方法时，希望将对象中的name属性打印出来。

当没有this的情况下，为了能够获取name属性，必须使用obj.name来获取。
弊端：当需要更改对象名obj为info的时候，对象内方法引用的obj.name都需要修改为info.name，非常的炸裂。

如果使用this对这个对象进行优化：修改对象名并不会影响对象内的方法，非常的紧致。

this的指向

this在全局作用域下永远指向window

显示绑定：给函数的调用指定this指向，可以通过call、apply、bind来指定this的指向

call

call(this指向，参数1，参数2，参数3…)

第一个参数是this即将指向的地方，后面的参数为参数列表：

假如传入的指定对象是null或者undefined，那这个显式绑定将会转为默认规则

apply

apply(this指向,[参数1，参数2，参数…])

第一个参数是this即将指向的地方，后面的参数为数组

bind

bind的用法和call类似：bind(this指向，参数1，参数2，参数3…)

唯一不同的是：bind返回一个函数，只有主动调用才会实现

内置函数

案例一：setTimeout

setTimeout内部是通过apply进行绑定的对象，并且绑定的是全局对象window

案例二：数组forEach

forEach内置函数中，this也是指向全局window。

如何让它指向指定的对象呢？：在第二个参数传入指定对象，此时this的指向就变成传入的对象了，就很美。

JavaScript中的函数可以当做一个类的构造函数来使用，通过new关键字。

使用new关键字来调用函数时，会执行以下操作：

  1. 创建一个全新的对象
  2. 这个对象会被执行prototype连接
  3. 这个新对象会被绑定到函数调用的this上
  4. 如果函数没有返回其他对象，表达式会返回这个新对象

规则优先级

new绑定 > 显示绑定bind > 隐式绑定 > 默认绑定

  1. 默认规则的优先级最低
  2. 显示绑定高于隐式绑定
  3. new绑定高于隐式绑定
  4. new绑定高于bind
