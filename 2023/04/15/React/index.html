渲染数据

JSX 中通过花括号{}来插入数据，相当于 Vue 中的双花括号{{}}插值语法。

渲染的内容： string、number、array、表达式。

会被忽略的内容：false、null、undefined。

JSX 中可以插入表达式，表达式代表有返回值的代码，例如：运算符表达式、三元表达式、函数调用、短路运算&&等。

React 核心内容

React 的两种组件

React 应用程序是一个一个组件构成，组件可以小到一个按钮，也可以大到整个页面。

React 允许你创建可重复使用的组件，每一个 UI 都是一个组件，组件定义一次，可以在多个地方重复调用。

函数组件就是 JavaScript 函数，但它必须符合两个规范：

  1. 函数名必须以大写字母开头
  2. 组件必须返回 JSX 或者 TSX 格式的内容
  3. 返回的内容必须只有一个根标签

React 渲染 JSX 的过程中，遇到小写的标签<section />就知道这是在渲染 HTML 标签，当遇到大写的标签<Profile />就知道它渲染的是一个组件

React18 中官方推荐，不要在一个组件中定义另外一个组件 ，更好的做法是将子组件抽离出来，如果想要实现组件间数据的共享，可以通过Props的形式传递。

class 类组件继承至 React 中Component类，在类组件实例的时候必须要调用super，内部通过render()方法渲染组件内容。

类组件中的构造函数constructor只做两件事：

  1. 通过 this.state 赋值对象的形式，来初始化类组件 state 对象。
  2. 为事件显示绑定 this 例如：this.btnClick = this.btnClick.bind(this)。

点标记语法

无论是函数组件还是类组件，都可以使用点标记语法操作组件。

这种写法很适合复杂组件，可以拓展子组件进行组合使用，更加具备语义划操作。

Hooks

hook 本身是一个函数。

⚠️注意：hook只能在组件的顶层使用，不可以放在语句中进行调用。

React内置的hooks：

  • State hook 创建响应式的状态
  • Context hook 创建上下文共用的状态
  • Ref hook 创建不需要响应式的状态（更新ref不会重新渲染组件）
  • Effect hook 允许连接外部系统

useState

允许向组件添加一个状态变量（响应式的变量）当数据发生改变的时候会重新渲染组件。

语法规则：const [something, setSomething] = useState(初始值)

useState 的特性：

惰性初始值

如果遇到初始值需要大量运算才能获取的，可以采用惰性初始化state，通过传入一个纯函数来实现初始值。

纯函数：永远只返回相同的结果，而不影响外部的状态的函数，例如下面的initCount方法。

set函数是异步的

set 函数在组件下一次渲染的时候，才更新状态变量，在 set 函数后面打印状态值，还是原来的旧值。

新旧值跳过机制

如果向 set 函数提供的值和旧值一样，React 会跳过组件的重新渲染（包括子组件）,这个是 React 内部做的一个优化方案。

修改引用类型原始值

set 函数在修改变量状态的时，要替换对象而不是改变现有的变量，对于修改的值，不会进行原值的合并处理，可利用扩展运算符来解决合并问题（可以使用第三方Immer库来解决这个问题）。

默认的批量处理能力

useState 是默认具有批量处理能力的，执行完所有的 set 函数队列，才会执行组件的重新渲染。

触发handleCLick事件时，会将 msg 和 count 的数据进行更新，组件会在msg和count的值一起更新后才重新渲染。

如果不想使用这种批量处理能力，想更新 msg 值后就重新渲染组件，组件再次渲染完成后，再继续执行 count 值的更新，React 提供了flushSync方法来实现组件的每次更新后重新渲染（需要从 react-dom 包中导入）

useEffect

useEffect 可以让你在函数组件中「执行副作用操作」，每次组件渲染都会执行一遍，相当于类组件中的「componentDidMount生命周期」

副作用包括：

  • DOM操作
  • 获取后台数据
  • 记录日志
  • 更新组件外部状态
  • 创建定时器
  • …
创建定时器

组件第一次渲染的时候，使用useEffect创建一个定时器，在组件卸载的时候，清理这个定时器。

useEffect中return的函数称为「清理函数」。

useEffect 的特性：

当作生命周期来用

因为函数组件中没有生命周期钩子，所以可以用useEffect替代类组件中的「生命周期钩子」。

  • useEffect 是异步函数，在「组件加载完毕」和「组件更新时」时触发。
  • useEffect 的回调函数，在「组件更新前」和「组件被卸载前」时被触发，常用于清理副作用。
钩子独立性

每一个useEffect钩子都是独立的，可以在组件中使用多个 useEffect 来分离关注点。

使用回调函数进行性能优化

可以通过useEffect的回调函数进行性能优化（清理副作用等）。

可控性

通过传入一个空数组来控制「useEffect」只在组件初次渲染的时候执行一次。

依赖处理

⚠️：如果「useEffect」中使用了某个响应式数据，一定要进行「数组依赖的处理」。

只有当依赖数组中的声明项发生了改变，才会重新执行 useEffect 的回调函数。

频繁修改某个响应式数据，可以通过回调函数改写

举例：在一个 effect 中，定时的去更新某个响应式状态，上面提到过只要 effect 中有响应式状态，就使用数组依赖处理，所以当 count 每次发生变化的，会重新执行 useEffect 的钩子，setInterval 会生成多个定时器 id，造成定时器的累加。

闪屏问题

useEffect是在渲染被绘制到屏幕之后执行的，是异步的。

uselayoutEffect是在渲染之后但在屏幕更新之前，是同步的。

大部分情况下我们采用 useEffect，性能好，但是当你需要在 useEffect 中操作 DOM，并且改变页面样式，就需要使用useLayoutEffect，否则会出现闪屏问题。
