vscode 编译 ts 文件

  1. 在项目中执行 tsc-init 命令，生成tscondig.json的配置文件。
  2. 在tsconfig.json文件中，outDir是用来指定编译的 js 文件存放位置。
  3. vscode 菜单 => 终端 => 运行任务 => TypeScript => tsc:监视-tsconfig.json。
  4. 每次修改 ts 文件，都会将 ts 文件自动编译为 js 文件到指定目录下。

TS 的数据类型

基础数据类型

TS 中内置的基础数据类型有：boolean、string、number、undefined、null、symbol、BigInt。

void 类型：常用于没有返回值的函数的类型。

any 类型：是所有类型的顶级类型，它会跳过 TS 类型检查机制，可以被其他类型任意分配。

unknown 类型：也是所有类型的顶级类型，也可以被其他类型任意分配。

undefined 和 null 是所有类型的子类型，在非严格模式下，都可以被其他类型所分配。

当你需要使用 any 的时候，不妨用 unknown 替代它。

数组

定义数组的三种方式：类型[] 、数组泛型、接口定义数组

元祖 Tuple

元祖是数组类型中的一种，表示已知元素和元素类型的数组，元素可以是不同数据类型。

类型断言

类型断言是指你比 ts 更懂这个数据是什么类型。

类型断言的两种方式：使用关键字 as 、**<类型>值**。

使用场景：

  • 将一个联合类型推断为一个类型
  • 将 any 类型断言成一个具体的类型
  • 将任意的一个类型断言成 any

类型断言只能够「欺骗」TypeScript 的编译器，无法避免运行时的错误，滥用类型断言可能会导致运行时报错。

小技巧

直接使用 const 定义的基础类型常量，是无法重新被赋值的。

如果用 let 定义的变量，也不能被重新赋值，就可以使用断言的方式： as const

函数重载

在 ES5 标准中，同名的函数和变量，后者会覆盖前者。

但是在 TS 中，同名的函数不会被覆盖，这会使得函数具有重载功能：同名的两个函数，传入不同类型的参数，达到返回不同类型的结果。

TS 的类

类的定义

在 ES6 中使用 class 关键字定义类。

通过 static 关键字可以定义静态属性和静态方法，这种属性和方法可以在类实例化之前被调用。

static关键字注意点

但是静态方法不可以访问类的其他类型的属性，只能访问静态属性，比如一下 print 方法不能访问 age 属性，但是可以访问 action 属性：

类的属性修饰符

类的属性和方法通过修饰符来约束访问和使用，三种修饰符：public 、protected、private。

public不用多说，使用 public 定义的属性和方法，在任意位置都可以使用。

protected：类内部、子类可以访问，外部不可以访问。

private：类的内部可以访问，子类和外部都不可以被访问。

类的继承

类和类之间可以通过 extend 来实现继承，父类被称为基类或者超类。

类的继承包含了两个关键字：constructor 和 super

  • 构造函数 constructor 会在类实例化的时候被调用。
  • super 是调用父类的构造函数，所以派生类一定要调用 super 方法。
  • 父类和子类如果有同名的属性和方法，会就近原则，执行子类里的方法。

类的多态

多态属于继承的一部分。

意思是：父类定义一个方法，但是父类不去实现它，让继承它的子类去实现，每一个子类都会有不同的表现。

简而言之：父类和子类有同名的方法，就近原则，执行子类里的同名方法。

抽象类

使用关键字 abstract 定义抽象类，抽象类的子类必须遵循父类的标准。

简而言之：抽象类里的属性和方法，派生类必须重写相同的属性和方法。

使用场景：定义一个父类作为类的标准，让派生类都必须按照这个父类的标准去重写属性或者方法。

类型别名

和接口的功能一样，类型别名也是约束数据的结构用的，使用关键字 type 定义：type 类型别名=约束规则

和接口不一样的是，类型别名可以用于 原始类型(基本数据类型)、引用类型、联合类型、元祖和指定值的别名。

类型别名也可以拓展，使用 & 符号

类型别名和接口的区别？

相同点

接口和别名都可以继承，接口和接口之间使用 extends，别名和别名之间使用&符号，接口和别名之间使用&符号

不同点

接口可以同名，接口同名会自动拓展，别名不可以同名。

接口不可以使用操作符，别名可以使用操作符(typeof、keyof 等)。
