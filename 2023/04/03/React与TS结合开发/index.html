children 类型限制

父子组件通信的时候，子组件内容分发是通过 props.children 实现的，那么怎么对 children 进行限定呢?

通过React.ReactNode对 children 进行限定，我们把 children 设置为可选属性，这样是否要进行内容分发都是可以的。

event 限制

事件对象在 React 中主要通过内置的对象类型来限制，常见的 div 的点击事件对象类型是：React.MouseEvent

父组件传递子组件一个点击事件handleClick

子组件定义 handleClick 方法的类型，这里的事件对象 ev 的类型是React.MouseEvent，并且它可接收一个泛型。

如果事件对象只能通过 button 元素点击才可以，可以给 MouseEvent 传递一个泛型，React.MouseEvent<HTMLButtonElement>，此时如果将 button 替换成普通的 div，将会报TS的错误

style 限制

父子组件传递也可以传递 style 样式，style 同样也可以被React.CSSProperties限制，当传递的样式属性或属性值不符合 css 的属性值的时候就会产生TS的类型错误提示

Hooks 函数限制

在函数组件中主要就是对 use 函数进行类型注解，常见的 hook 类型有以下几种：

  • useState -> 联合类型，对象字面量类型

  • useRef -> 泛型标签类型

    如果不传递泛型标签，使用的时候会报错，也可以通过?可选链 !非空断言来阻止 TS 的报错

    可选链属于类型保护，而非空断言相当于是欺骗 TS(慎用)

  • useEffect -> 自动推断类型

类组件类型限定

在这里只是作为拓展，也要知道怎么对类组件进行类型的限制。

主要就是给继承的类Component传递泛型，Props和State，这样可以实现父子通信的数据进行类型限制，又可以对内部的 state 进行类型限制。

  • RouteObject 内置类型 -> 用来限制路由表 routes
